# DS Beamforming — 延迟求和波束成形模块

## 简介

本模块实现了基于 **GCC-PHAT 时延估计**的**延迟求和（Delay-and-Sum）波束成形**算法，适用于 4 麦克风线性/平面阵列的语音增强场景。通过估计各通道信号相对参考麦克风的时延，对多路信号进行时延对齐后叠加，从而在目标方向形成增益、抑制非目标方向的噪声干扰。

---

## 文件说明

| 文件 | 说明 |
|------|------|
| `ds_beamforming.c` | 算法核心实现，包含初始化、TDOA 估计、延迟求和三个模块 |
| `ds_beamforming.h` | 对外接口声明 |

---

## 接口说明

### `void ds_beamforming_init()`

初始化波束成形模块，需在调用 `ds_beamforming_process()` 之前调用一次。

- 初始化 256 点 FFT 句柄
- 预计算 128 点汉明窗系数

---

### `void ds_beamforming_process(short input_channels[][128], short *out)`

逐帧处理多路麦克风输入，输出波束成形后的单路语音。

| 参数 | 类型 | 说明 |
|------|------|------|
| `input_channels` | `short[][128]` | 4 路麦克风输入，每路 128 个采样点（16位PCM） |
| `out` | `short *` | 输出缓冲区，输出 128 个采样点（16位PCM） |

> 首帧调用时模块完成缓冲区初始化，输出静音帧；从第二帧起正常处理并输出。

---

## 算法流程

```
4路麦克风输入（128帧）
        │
        ▼
  缓冲区管理（256点滑动窗）
        │
        ├─── [第一半帧] TDOA 时延估计
        │          │
        │     加汉明窗 + 256点 FFT
        │          │
        │     GCC-PHAT 互相关（各通道 vs 参考通道0）
        │          │
        │     IFFT → 互相关峰值搜索（±4帧范围）
        │          │
        │     输出各通道时延 best_delay_index[]
        │
        ├─── [第一半帧] 延迟求和
        │          │
        │     按 best_delay_index 对齐各通道
        │          │
        │     梯形加权叠加 → 输出前64帧
        │
        ├─── [第二半帧] 重复 TDOA + 延迟求和 → 输出后64帧
        │
        ▼
  单路增强语音输出（128帧，16位PCM）
```

---

## 关键参数

| 宏定义 | 值 | 说明 |
|--------|----|------|
| `N_CHANNEL` | 4 | 麦克风通道数 |
| `REFER_CHANNEL` | 0 | 参考通道索引 |
| `N1_LEN` | 64 | 每次处理的帧长（半帧） |
| `N2_LEN` | 128 | 汉明窗长度 / 每帧采样点数 |
| `N4_LEN` | 256 | FFT 点数 |
| `MARGIN_FRAMES` | 4 | 时延搜索范围（±4帧） |

## 使用示例

```c
#include "ds_beamforming.h"

// 初始化（程序启动时调用一次）
ds_beamforming_init();

// 每次收到新的麦克风数据时调用
short mic_input[4][128];   // 4路麦克风输入
short output[128];         // 增强后的单路输出

// ... 填充 mic_input 数据 ...

ds_beamforming_process(mic_input, output);

// output 即为波束成形后的语音数据
```

---

## 注意事项

1. 输入数据格式为 16 位有符号 PCM，内部转换为浮点后处理，输出时还原为 16 位。
2. 时延搜索范围为 ±4 帧，适用于麦克风间距较小（近场拾音）的场景，若阵列间距较大需调整 `MARGIN_FRAMES`。
3. 参考通道固定为通道 0，其他通道相对通道 0 进行时延对齐。
4. 模块内部使用全局变量，**非线程安全**，多实例使用需自行封装。
